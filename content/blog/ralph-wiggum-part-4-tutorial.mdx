## Handling Errors (What Went Wrong)

Ralph hit a couple of snags. Here's how they were resolved:

### Error 1: Missing Import (Iteration 9)

**Error:**
```
[Iteration 9] Lint error...
Module not found: Can't resolve '@/convex/_generated/api'
```

**What happened:** Ralph tried to import from Convex before the types were generated.

**How Ralph fixed it:**
```
[Iteration 10] Detected missing Convex types
[Iteration 10] This is expected - types generate on dev server start
[Iteration 10] Import is correct, marking task complete
```

**Learning:** Ralph understood this was a development environment issue, not a code bug.

### Error 2: State Update Logic (Iteration 12)

**Error:**
```
[Iteration 12] Quiz not advancing to next question...
Previous button disabled when it shouldn't be
```

**What happened:** State update timing issue with React's batching.

**How Ralph fixed it:**
```
[Iteration 13] Analyzing state flow...
[Iteration 13] Issue: checking index before state update completes
[Iteration 13] Fix: use callback form of setState and memoize handlers
[Iteration 13] Quiz navigation working correctly
```

**Solution Ralph implemented:**

```typescript
// Before (problematic)
const handleNext = () => {
  setCurrentQuestionIndex(currentQuestionIndex + 1);
  // Bug: checking old value
};

// After (fixed)
const handleNext = useCallback(() => {
  if (isLastQuestion) {
    // handle completion
  } else {
    setCurrentQuestionIndex((prev) => prev + 1);
  }
}, [isLastQuestion]);
```

## Key Lessons Learned

### What Worked Well

1. **Real project context**: Building for an actual platform meant realistic requirements and patterns
2. **Existing UI components**: Ralph leveraged shadcn/ui components already in the project
3. **Clear scoring system**: Weighted scoring was well-defined in specs, making implementation straightforward
4. **Atomic tasks**: Each component was a single task, allowing clean commits
5. **Pattern matching**: Ralph picked up on existing code conventions automatically

### What Could Be Improved

1. **Test coverage**: We didn't include test requirementsâ€”next time add unit tests for scoring logic
2. **Accessibility audit**: Should verify screen reader experience
3. **Analytics events**: Could add PostHog tracking for funnel analysis
