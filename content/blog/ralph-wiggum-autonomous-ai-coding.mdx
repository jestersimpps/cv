## What is Ralph Wiggum?

Ralph Wiggum is an official Claude Code plugin that turns your AI assistant into an autonomous coding agent. Named after The Simpsons' lovably persistent character, it embodies a simple but powerful philosophy: **Iteration > Perfection**.

As [Geoffrey Huntley](https://github.com/ghuntley/how-to-ralph-wiggum), the technique's pioneer, puts it: "Ralph is a Bash loop." That's it. A `while true` that repeatedly feeds Claude the same prompt until the job is actually done.

Here's the magic:

1. You give Claude Code a task with clear completion criteria
2. Claude works on it and tries to exit when "done"
3. A Stop hook intercepts the exit and checks if it's *really* done
4. If not complete, the same prompt is fed back in
5. Claude sees its previous work in the files and git history
6. The loop continues until genuine completion

**The key insight?** The prompt never changes—but Claude's work persists. Each iteration builds on the last, creating a self-referential feedback loop where Claude improves its own output until it actually works.

## The Three-Phase Methodology (How the Pros Do It)

Simple tasks? Just run `/ralph-loop` and let it rip. But for serious projects—the kind that used to take weeks—professionals use a structured three-phase approach:

**Phase 1: Requirements (The Human Part)**

This is where you earn your paycheck. Spend 30+ minutes talking through requirements with Claude before anything gets built. Define specs in dedicated files. Create acceptance criteria that can be automatically verified.

> **Why this matters:** Vague requirements produce vague results. "Make it better" isn't testable. "All tests pass with 80%+ coverage" is.

**Phase 2: Planning (AI Analyzes, You Approve)**

Ralph analyzes your requirements against the existing codebase, identifies gaps, and generates a prioritized implementation plan. No code written yet—just a TODO list with dependencies mapped out.

**Phase 3: Building (The Overnight Shift)**

This is where the magic happens. Ralph picks up tasks one by one, implements them, runs tests, fixes failures, commits working code, and moves to the next task. Rinse and repeat until the plan is complete.

You wake up to a git history full of clean, tested commits. Review them over coffee.

## Get Started Now

The learning curve is smaller than you think. Install the plugin, run your first loop on something small, and experience the shift firsthand.

```bash
/plugin install ralph-loop@claude-plugins-official
```

Then try something simple:

```bash
/ralph-loop "Add comprehensive tests to src/utils.ts. Run tests after each addition. Output <promise>DONE</promise> when coverage exceeds 80%." --completion-promise "DONE" --max-iterations 10
```

Watch it work. Review the results. Then scale up.
